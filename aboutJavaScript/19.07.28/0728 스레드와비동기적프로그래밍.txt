스레드 : 실제 작업을 수행하는 것

자바스크립트 - 실행환경은 멀티스레딩, 개발자가 코드로 제어할 수 있는 스레드를 한 개로 제한하여 싱글스레드 프로그래밍.....
=> 스레드의 블록킹 발생 가능
=> 비동기적 프로그래밍! (Call Stack 호출 순서와 다르게 실행되는 프로그래밍-동기적 흐름과 다름)

##비동기 처리해야할 작업##
⊙ 파일 읽기, 쓰기
⊙ AJAX 통신
⊙ DOM 이벤트 처리
⊙ 일정시간 후 동작작업...

==> 메인스레드에 맡기지 않고, 다른 방법(JS 비동기 API)을 이용해 처리하도록 한다

ex) setTimeout..

비동기적 프로그래밍 동작방식;
	1. Call Stack에 웹 api를 이용한 함수가 호출되어 들어옴
	2. 웹api (비동기처리를 위한 api)로 실행문이 넘어감
		웹api 종류; DOM, AJAX, setTimeout 등..
	3. 그 다음 코드가 읽혀서 call stack에 들어옴
	4. 들어왔던 코드가 넘어가 실행되고 난 뒤, 웹 api에 들어갔던 내용이 실행되는데 . . ..
	   이 때, task queue로 갔다가.. 빠져서 call stack으로 넘어간다.

=> 비록 자바스크립트가 싱글스레드프로그래밍이지만, js API를 이용해 동시성작업들을 처리하므로
     스레드의 블록킹 없이 (non-blocking) 동작할 수 있음
